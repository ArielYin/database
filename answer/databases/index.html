Databases

Here is a summary of design decisions for databases:

Database.java
Table.java (no change)
Record.java (no change)
Makefile

A database is stored as a collection of table files in a particular folder. A Database object is created by passing the folder path to the constructor. The Database object then finds the files and causes them to be loaded into tables:

class Database {
    private Map<String,Table> tables;

    Database(File folder) {
        tables = new TreeMap<>();
        File[] files = folder.listFiles();
        for (File file : files) {
            .... new Table ....
        }
    }
}

Table names: I have decided not to store the name of a table inside the table object. That means, when iterating, it is more convenient to iterate through the table names rather then through the table objects themselves, otherwise the names would be hard to get at.

The methods are fairly obvious, and there is no testing, because it is mostly I/O and will get tested well enough by being used.
